# 路线图

## 阶段状态
- [x] Phase A：观测层（消息识别 + 指标采集）
- [x] Phase B：静态裁剪原型（固定阈值 + 占位符 + 手动恢复）
- [x] Phase C：自动窗口化（滚动驱动 + rAF 调度 + 锚点校正）
- [x] Phase D：配置与控制（popup/options + storage）
- [x] Phase E：稳定性加固（fail-open + 重绑定）

## 下一步
1. 加入针对真实长会话的参数自适应策略。
2. 补充更细粒度指标（长任务、输入延迟估算）。
3. 基于真实使用反馈收敛默认阈值。

## 风险
1. ChatGPT DOM 结构变化可能导致选择器失效。
2. 特殊内容块（复杂富文本）恢复后高度波动可能引起轻微跳动。
3. 极快滚动时恢复频率上升，可能出现短暂占位符闪现。
